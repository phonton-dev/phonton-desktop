# Phonton Desktop v0.2.10

## Fixes

- **Sidecar RPC from the webview** — `phonton serve` health checks used GET (worked), but JSON-RPC `ping` uses POST and was blocked by missing CORS in the CLI. Desktop now proxies `/health` and `/rpc` through Tauri (Rust + reqwest), so setup connects without waiting on a CLI npm update.
- **Stale port cleanup** — if something is listening on `:47831` but RPC fails, setup kills the listener and respawns.
- **Native spawn hardening** — `phonton.exe serve` runs with vendor `current_dir`; desktop verifies the child is still alive after spawn before declaring success.
- **Setup UX** — install success + sidecar failure now shows **CLI offline** instead of **CLI install error**.
- **Exe launch spec** — after npm install/bootstrap, desktop prefers `vendor/phonton.exe` for sidecar spawn when verified.

## Companion CLI

`phonton-cli` **0.20.2** adds CORS headers on all serve routes so browser/webview fetch works without the desktop proxy (recommended upgrade: `npm install -g phonton-cli@latest`).
